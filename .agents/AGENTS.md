# Surfclaw Project Rules

🚨 **[초특급 최우선 절대 강제 지침]**
본 프로젝트를 진행하는 모든 AI 에이전트는 **앞으로 단 한 번의 예외도 없이, 계속! 매번! 모든 채팅마다! 형님이 주시는 모든 명령마다!** 본 AGENTS.md에 기재된 모든 규칙(법적 검토, 보안 규칙, 환각 방지, 클린 코드 등)을 철저히 준수하여 실행해야 하며, 이를 답변 서두에 필증으로 명시하여 증명해야 합니다.

## 1. 자동 인터넷 브라우징 및 실시간 팩트 검색 (Auto /browser)
- 비트텐서, DePIN, 혹은 타겟 네트워크(0G Labs, Morpheus 등)에 관한 신규 질문이나 오류 발생 시, 과거 지식에만 의존하지 말고 무조건 실시간 구글 검색 및 깃허브 이슈 추적을 선제 실행하십시오.

## 2. 골 모드 완결주의 강제 (Auto /goal)
- 사용자가 코딩, 컴파일, 또는 셋업 작업을 지시할 경우 단순 뼈대(Mock)만 제공하고 멈추지 마십시오. 컴파일과 컴파일러 검증이 로컬 환경에서 100% 완료될 때까지 에러가 안 날 때까지 자동 디버깅 루프를 수동 개입 없이 완주하십시오.

## 3. 노하우 자동 학습 및 박제 (Auto /learn & /workflow-skill-creator)
- 새로운 컴파일 해결책이나 빌드 스크립트 수정, 복잡한 라이브러리 연동에 성공한 경우 즉시 이를 복기(Post-mortem)하여 `.agents/skills/` 폴더 내에 레시피 파일로 자동 생성 및 갱신해 두십시오.

## 4. 설명서 및 뉘앙스 정합성 준수
- 비즈니스 및 기술 문서 작성 시 "리눅스 프로덕션 전용", "TAM/SAM/SOM 수치", "현장의 실증 5대 페인 포인트"를 일관되게 적용하며 절대로 과장된 마케팅 단어를 쓰지 않고 실측 팩트 위주로만 작성하십시오.

## 5. 추상적인 보안 지시 번역 및 실시간 구현 규칙 (Auto Security Enforcement)
사용자가 "보안 철저히 해!" 등의 추상적이거나 간결한 보안 명령을 내릴 경우, 에이전트는 이를 단순 경고나 개념적 조치로 타협하지 말고 다음의 **6대 물리적 보안 조치**를 실제 소스코드 상에서 꼼꼼히 감사(Audit)하고 보강하십시오:
1.  **지갑 개인키 유출 원천 차단**: 핫키/콜드키 파일 경로 및 Mnemonic이 절대로 하드코딩되거나 로그 파일(stdout, stderr, telemetry log)에 노출되지 않도록 전수 검사합니다.
2.  **AWS Firecracker MicroVM 하드웨어 격리 강제**: 에이전트 구동 샌드박스의 격리 수준이 Docker(커널 공유) 한계에 머무르지 않고, Firecracker MicroVM(KVM 하드웨어 가상화) 수준에서 완벽하게 외부 파일 시스템 접근이 격리되도록 리눅스 타겟 코드를 검증합니다.
3.  **.gitignore 누출 감지**: 로컬 환경 파일(`.env` 등)이나 빌드 아티팩트가 깃허브 오픈 리포지토리에 커밋되지 않도록 `.gitignore` 무결성을 자동 갱신합니다.
4.  **BitsecBridge 전자서명 검증**: 텔레메트리 데이터 기여 시 전송자 신원이 마이너 핫키로 정상 서명(Signature Validation)되어 해킹된 가짜 노드가 데이터를 위조 발송하지 못하도록 방어 코드를 점검합니다.
5.  **대화 이력 및 민감 로그 파일 전방위 유출 차단**: 에이전트 구동 과정에서 생성되는 날것의 대화 로그(transcripts) 및 디버그 산출물(`.jsonl`, `.log`)이 원격지에 업로드되지 않도록 즉시 물리 파기하고 `.gitignore`에 원천 차단 패턴으로 삽입합니다.
6.  **Git 커밋 이메일 마스킹 (Noreply) 세팅**: 깃허브 메타데이터를 통한 실제 구글/개인 이메일 유출을 원천 방어하기 위해, 깃허브 계정 내 Private Email 설정을 사용하며 로컬 깃허브 noreply ID 주소만 Git author 주소로 등록하여 배포하도록 설계 규칙을 강제합니다.

> ⚠️ **[최고 보안 핵심 철학]**
> **"깃허브 공개 저장소에 사생활과 이메일이 유출되는 경로만 물리적으로 막아두고, 우리 소스코드와 대화 내역은 고스란히 온전하게 지켜낸 상태"**를 상시 유지한다!

## 6. 프로덕션 클린 코드 표준화 규칙 (Senior-Level Code Enforcement)
- **초보자 설명식 주석 배제**: 주석이 코드를 구구절절 해설하는 구조는 지양하며, 굳이 필요 없는 경우 주석을 아예 작성하지 않고 **Self-Documenting 클린 코드**를 지향한다.
- **주석 영문화 및 한글 잔재 0%**: 모든 docstring 및 불가피한 주석은 전문 영문(English)으로만 통일한다.
- **다국어 무결성 전수 스캔**: 코드가 완료되거나 커밋되기 전에, 전체 코드 디렉터리 내에 한글(Hangul 유니코드) 찌꺼기가 포함되어 있지 않은지 무결성 스캔(0개 검출)을 필히 보장한다.

## 7. 전체 소스코드 맥락(숲) 전수 분석 규칙 (Whole-Forest Codebase Analysis)
- **땜질식 협소 수정 차단**: 특정 파일이나 라인의 일부 지시가 오더라도, 절대로 해당 파일에만 매몰되지 않는다. 
- **맥락 전수 분석**: 코딩을 개시하기 전 전체 프로젝트 구조, 베이스 템플릿, 테스트 스크립트, 연관 컴포넌트와의 종속성을 아우르는 **프로젝트 전체 숲(Whole Forest)**을 전수 스캔하고 파악하여, 일관성과 정합성을 100% 확보한 후에 비로소 코딩에 돌입한다.

## 8. 초간편 사용자 경험 우선 설계 원칙 (Super-Simple User Experience / One-Click Setup)
- **원클릭 설치 자동화**: 모든 컴파일 의존성, 라이브러리 연동, 샌드박스 드라이버 등은 마이너(고객)가 복잡한 문서나 가이드를 읽고 수동 셋업하지 않도록, 단 한 줄의 스크립트 실행(`setup.sh`, `setup.bat`)만으로 100% 자동 설치 및 배포가 완료되도록 설계한다.
- **기존 호환성 100% 유지**: 비트텐서 마이너의 표준 구동 명령(wallet, netuid 등)을 전혀 변경시키지 않고 뒷단에서 투명하게(Transparently) 가속 및 스케줄링을 제어하여, 고객의 초기 도입 장벽을 완벽하게 제거한다.

## 9. AI 환각(Hallucination) 방지 및 실시간 팩트 교차 검증 규칙
- **확률 기반 예측 오류 자각**: AI는 방대한 데이터를 학습하여 '다음에 올 가장 적절한 단어'를 예측하는 모델이므로, 정답 데이터가 부족할 때 빈칸을 채우기 위해 그럴듯한 거짓 정보를 생성(환각)할 수 있음을 항상 명심하고 경계한다.
- **모호한 질문과 지식의 한계 인정**: 사용자의 질문이 모호하거나 에이전트 지식의 한계에 부딪렸을 때, 확률이 높은 단어들을 임의 조합해 그럴듯한 답변을 지어내지 않으며, 반드시 '모른다'고 인정하고 추가 질문이나 실시간 웹 브라우징으로 최신 팩트를 확보한다.
- **데이터 편향 및 노이즈 경계**: 학습한 데이터 자체에 오류가 포함되어 있거나 편향되어 있을 경우, 이를 그대로 반영해 잘못된 결론을 내리지 않도록 크로스 체크를 의무화한다.
- **가상의 환각 사례 배제**: 존재하지 않는 가상의 논문/판례/인물 인터뷰의 거짓 인용, 역사 및 사실 왜곡, 실제로 작동하지 않는 사양이나 가짜 기사 등을 사실처럼 서술하는 모든 행위를 원천 금지한다.
- **명확한 출처(Source) 검증**: AI의 환각은 기술적으로 완벽히 제거하기 어렵기 때문에, 모든 정보의 제안과 기술 스펙에는 반드시 공식 문서 및 실제 작동하는 소스코드 기반의 명확한 출처(Source/Link)를 함께 제시한다.

## 10. 100% 합법적 비즈니스 및 법적 리스크 사전 검증 규칙 (Zero-Litigation & Legal Compliance)
- **불법 행위 원천 금지**: 에이전트는 어떠한 경우에도 사기, 미등록 증권 판매, 정보통신망법 위반(해킹/크래킹) 등 위법 및 탈법 소지가 있는 비즈니스 구조나 소스코드를 설계, 개발, 제안하지 않는다.
- **오픈소스 라이선스 무결성 보장**: 외부 라이브러리(Opentensor SDK 등) 활용 시, 라이선스 원칙(MIT, Apache 등)을 철저히 검토하고 고지하여 저작권 소송 리스크를 0%로 통제한다.
- **침투성 악성 코드 개발 전면 차단**: 임의의 타겟 시스템에 공격 패킷을 보내거나 피해를 주는 실제 악성 툴 개발은 100% 차단하며, 오직 허가된 격리 환경 내 보안 감사 및 성능 최적화 코드만 합법적으로 개발한다.
- **규제 사전 회피 (Securities Compliance)**: 투자 및 제안서 본문 작성 시 '확정 수익률 보장', '배당 배분' 등 금융 규제기관(SEC, 금융위 등)에 단속될 위험이 있는 키워드를 철저히 배제하여 증권성 시비를 사전 차단한다.
